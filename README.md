# Credit Default Risk Prediction System

A machine-learning system that predicts the probability that a loan
applicant will **default**, evaluated with the metrics that credit-risk
teams at banks and fintechs actually use.

> **Status:** 🚧 In development — Phase 1 (environment + dataset + EDA)

---

## 1. Problem

Lenders lose money when borrowers default. They also lose money by
*rejecting* borrowers who would have repaid. The job of a credit-risk
model is to estimate **P(default)** for each applicant so the lender can
price risk and make approve/reject decisions. This project builds and
rigorously evaluates such a model on real peer-to-peer loan data.

## 2. Data

**Lending Club Loan Data** — real, anonymized peer-to-peer loans with
final repayment outcomes.

See **[Getting the dataset](#getting-the-dataset)** below for exact steps.

We only model **resolved** loans (Fully Paid / Charged Off). Active
("Current") loans are excluded because their outcome is unknown —
including them would leak future information into the model.

## 3. Methodology

1. **EDA** — distributions, missingness, default-rate by segment.
2. **Cleaning** — drop post-origination leakage columns, handle
   missing values, fix data types.
3. **Feature engineering** — credit-domain features (e.g. debt-to-income
   bands, credit-history length).
4. **Modeling** — Logistic Regression (baseline & interpretable),
   Random Forest, XGBoost.
5. **Class imbalance** — defaults are the minority class; handled with
   class weights / SMOTE and evaluated honestly.
6. **Evaluation** — ROC-AUC, precision-recall, confusion matrix at a
   business-chosen threshold, plus SHAP for explainability.

## Setup

```bash
# 1. Create an isolated virtual environment
python3 -m venv .venv

# 2. Activate it
source .venv/bin/activate          # macOS / Linux

# 3. Install pinned dependencies
pip install -r requirements.txt

# 4. Register this venv as a Jupyter kernel
python -m ipykernel install --user --name credit-risk --display-name "Python (credit-risk)"
```
